## ⚠️ 가장 중요 — 앱 서명 키스토어

**절대 잃어버리면 안 됨. 잃어버리면 앱 업데이트 영구 불가.**

### 키스토어 생성 (아직 없는 경우)
```bash
keytool -genkey -v -keystore ~/paymoa-release.jks \
  -keyalg RSA -keysize 2048 -validity 10000 \
  -alias paymoa
```
- 비밀번호 반드시 기록해둘 것
- 파일 안전한 곳에 백업 (Google Drive, 외장하드 등)

### android/key.properties 파일 생성
```
storePassword=여기에비밀번호
keyPassword=여기에비밀번호
keyAlias=paymoa
storeFile=/Users/사용자이름/paymoa-release.jks
```

### android/app/build.gradle 서명 설정 확인
release 빌드에 signingConfig가 연결되어 있는지 확인.

### AAB 빌드
```bash
flutter build appbundle --release
```
결과물: `build/app/outputs/bundle/release/app-release.aab`

## Play Console 설정 순서

### 1. 앱 만들기
- 앱 이름: 페이모아
- 기본 언어: 한국어
- 앱 또는 게임: **앱**
- 유료 또는 무료: **무료** (인앱결제로 수익화)

### 2. 앱 콘텐츠 설정 (중요)

#### 개인정보처리방침
- **필수** — URL 입력해야 함
- 앱 내 `/privacy` 라우트에 화면 있음
- 하지만 웹 URL이 필요 → GitHub Pages나 노션 공개 페이지에 올려야 함
- 아니면 Firebase Hosting으로 배포

#### 광고
- "앱에 광고가 포함되어 있나요?" → **아니요** (지금은 광고 코드 없음)
- 나중에 광고 추가 시 **예**로 변경

#### 앱 액세스 권한
- 특별한 로그인이 필요한 경우 테스트 계정 제공
- 카카오/구글/애플 로그인 → 테스트 구글 계정 하나 만들어두기

#### 콘텐츠 등급
- 설문 성실히 작성
- 폭력/성인 콘텐츠 없음, 비즈니스 앱
- 결과: 전체 이용가(Everyone) 예상

#### 타겟 고객층
- 만 13세 이상
- 어린이 대상 앱 아님 → **아니요** 선택

#### 데이터 보안 (매우 중요 ⚠️)
수집하는 데이터 정직하게 신고해야 함:
| 데이터 종류 | 수집 여부 | 이유 |
|--|--|--|
| 이름 | ✅ | 회원 프로필 |
| 이메일 | ✅ | 로그인 (구글) |
| 사용자 ID | ✅ | Firebase Auth |
| 기기 ID | ✅ | Firebase |
| 앱 활동 | ✅ | 근무 기록, 급여 데이터 |
| 재무 정보 | ✅ | 급여 관련 데이터 |
| 위치 정보 | ❌ | 미사용 |
- 제3자 공유: Firebase (Google) — 반드시 표시
- 데이터 암호화 전송: ✅ (HTTPS/TLS)
- 데이터 삭제 요청 가능 여부: ✅ (회원탈퇴 기능 있으면)

### 3. 스토어 등록정보

#### 앱 이름
페이모아 - 알바 급여 계산기

#### 간단한 설명 (80자 이내)
사장님과 알바생을 위한 스마트 급여 관리 앱

#### 자세한 설명 (4000자 이내)
예시:
```
페이모아는 자영업자(사장님)와 알바생을 위한 급여 관리 앱입니다.

[사장님 기능]
• 알바생 근무 스케줄 관리
• 자동 급여 계산 (시급, 주휴수당, 야간수당 포함)
• 급여대장 엑셀 파일 다운로드
• 매장별 인력 관리

[알바생 기능]
• 내 근무 일정 확인
• 예상 월급 실시간 계산
• 근무 기록 조회

간편 로그인: 카카오, 구글, 애플
```

#### 스크린샷
- 핸드폰 스크린샷: **최소 2장, 최대 8장** 필요
- 권장 크기: 1080 x 1920 또는 1080 x 2340
- 삼성 SM G998N에서 직접 캡처해서 올리면 됨
- 주요 화면: 로그인, 매장 목록, 근무 캘린더, 급여 계산, 급여대장

#### 앱 아이콘
- 512 x 512 PNG (투명 배경 없음)
- 현재 앱 아이콘 있으면 그대로 사용

#### 특집 그래픽 (선택)
- 1024 x 500 JPG/PNG
- 없어도 출시 가능

### 4. 출시 트랙 선택

**내부 테스트 → 비공개 테스트 → 공개 테스트 → 정식 출시** 순서 권장

처음이라면:
1. **내부 테스트**로 AAB 업로드
2. 본인 구글 계정으로 테스트
3. 문제 없으면 **정식 출시**로 진행

### 5. 가격 설정
- 앱 자체: **무료**
- 인앱 결제: 나중에 결제 연동 후 추가
